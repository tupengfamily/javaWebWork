## Why

当前「数据管理」页面（`/admin/data`）仅提供基础的小说列表浏览和单条删除功能，与「任务管理」页面存在功能重叠（站点状态卡片），且缺乏数据质量监控、批量操作和数据导出能力。随着站点从 3 个扩展到 6 个、数据量持续增长，管理员需要更高效的数据治理工具来维护数据质量、清理脏数据、导出分析数据。

## What Changes

### 现状评估（有价值的部分）

| 功能 | 价值 | 建议 |
|------|------|------|
| 站点状态卡片 | ❌ 与任务管理页重复 | 移除，避免页面臃肿 |
| 小说列表（搜索/分页） | ✅ 核心功能 | 保留并增强 |
| 单条删除 | ✅ 有但太弱 | 扩展为批量操作 |
| 排行记录弹窗 | ✅ 有但信息不足 | 增强展示 |

### 增强功能

- **数据质量仪表盘**：显示每个站点的数据完整性指标（缺封面、缺简介、缺作者的小说数量），帮助管理员快速发现数据问题
- **批量操作**：支持多选删除、批量导出，提高管理效率
- **数据导出**：支持将小说列表和排行记录导出为 CSV，方便离线分析
- **高级筛选**：按数据完整性（有/无封面、有/无简介）、更新时间范围等维度筛选
- **小说详情增强**：排行记录弹窗增加趋势迷你图，直观展示排名变化

### 不建议做的功能

| 功能 | 原因 |
|------|------|
| 数据手动编辑 | 爬虫数据应由爬虫维护，手动编辑会被下次爬取覆盖 |
| 数据对账/校验 | 当前规模（<1000 本）人工即可管理，自动化校验 ROI 低 |
| 数据备份/恢复 | 已有 docker volume + MySQL，不属于应用层职责 |

## Capabilities

### New Capabilities
- `data-quality`: 数据质量仪表盘，展示各站点数据完整性指标和问题数据清单
- `data-export`: 数据导出功能，支持小说列表和排行记录的 CSV 导出
- `batch-operations`: 批量操作，支持多选删除和批量导出

## Impact

### 受影响代码
- **前端**：`CrawlerData.vue` — 重写页面布局，移除站点状态卡片，增加质量仪表盘和批量操作
- **前端**：新增 `api/admin-data.ts` — 数据管理和导出 API 封装
- **后端**：`AdminNovelController.java` — 新增批量删除、数据统计、导出接口
- **后端**：新增 `AdminDataController.java` — 数据质量统计专用接口

### 依赖
- 无新增第三方依赖
- CSV 导出使用 Java 内置 `PrintWriter` + `response.setContentType("text/csv")` 即可

### API 变更
- `GET /api/admin/data/quality` — 新增，返回各站点数据完整性统计
- `DELETE /api/admin/novels/batch` — 新增，批量删除小说
- `GET /api/admin/novels/export` — 新增，CSV 导出
- `GET /api/admin/novels/{id}/records/export` — 新增，排行记录 CSV 导出
